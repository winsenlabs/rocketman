# Rocketman — agent guide

This repo is **Rocketman**, an in-repo HTML project hub + a Claude skill stack that walks a
product from idea to production. It dogfoods itself: its own roadmap, spec, decisions, and
debugging trails live in [`PM/index.html`](PM/index.html).

## The golden rule

> **Edit the data, never the HTML. Then rebuild.**

`PM/index.html` is generated by `rocketman build` from `PM/data/*.json`. A PreToolUse hook
(`.claude/hooks/guard-generated.sh`) blocks direct edits to it. The full data schema and the
single-writer rule for agent fleets are in [`PM/CLAUDE.md`](PM/CLAUDE.md) — read it before
changing any data.

## How the pieces fit

- `engine/` — the reusable hub engine: `build.mjs` (pipeline), `app.js` (renderer), `hub.css`.
- `bin/rocketman.mjs` — the CLI: `init · build · check · new · doctor · serve · relay`.
- `PM/data/` — the source of truth (core, tasks, spec, content) + `PM/comms/` (the agent relay).
- `.claude/skills/` — the Rocketman Track skill stack; shared rules in
  [`.claude/skills/CONVENTIONS.md`](.claude/skills/CONVENTIONS.md).

## Skill routing

When the user's request matches a skill, invoke it via the Skill tool. When in doubt, invoke it.

- New idea / "is this worth building" → `/rm-ideate`
- Write the spec / PRD → `/rm-prd`
- Plan / break into tasks / roadmap → `/rm-plan`
- Build / work the board / allocate to agents → `/rm-build`
- Verify / run checks / "is this ready" → `/rm-verify`
- Human test / QA → `/rm-test`
- Launch / ship / cut a release → `/rm-launch`
- Post-launch feedback / changes → `/rm-iterate`
- Multiple agents, handoffs, autopilot → `/rm-relay` (pair with `/loop`)
- Record a decision / ADR → `/rm-decision`
- Debug / "why is this broken" → `/rm-debug`
- Status / standup / "where are we" → `/rm-brief`
- Sync the hub from git → `/rm-sync`
- Mirror to Linear/FanDesk/etc. → `/rm-mirror`
- Set up Rocketman in a repo → `/rm-init`

Every skill follows `.claude/skills/CONVENTIONS.md`: plan-mode behavior, the decision-brief
question format, the Completion Status Protocol (DONE / DONE_WITH_CONCERNS / BLOCKED /
NEEDS_CONTEXT), and the builder-to-builder voice.

## After any change

```bash
rocketman build       # regenerate PM/index.html + run the doctor
```

Commit `PM/data/` and `PM/index.html` together with the related code.
